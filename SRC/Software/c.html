.c
fichiers sources
.h
fichiers d'en tête, descriptif des fonctionnalités offertes (prototype des fonctions, type ...) par le fichier .c associé
#pragma once
a utiliser dans chaque fichier .h en en tete
#include fichier.h
va copier l'integralite du code .c associé dans le fichier qui fait l'appel
utiliser "fichier.h" si le fichier.h est local sinon < fichier.h > pour un fichier de l'api C
Une fois le code .c écrit, il doit d'abord être compilé, pour pouvoir ensuite être exécuté Pour la compilation et l'exécution on peut se passe d'un IDE :
gcc nomFichier.c -o nomPourExecutable
pour compiler le code .c et en créer un fichier executable
clang nomFichier.c -o nomPourExecutable
pour compiler le code .c et en créer un fichier executable sur mac versions supérieures a OSX
-std=c99
option de compilation, pour specifier le type de C employé
./nomExecutable
pour exécuter le code C compilé
-lm
a rajouter pendant la compilation pour utiliser la librairie maths
Compilation avec dépendances :
gcc -c -o fichier.c fichier.o
creer le fichier objet associé, sans les liens entre fichier (a faire pour chaque .c dans un premier temps)
gcc -o nomExecutable liste des fichiers .o requis
faire les liens entres les différents fichiers, dans un second temps pour obtenir l'executable (bibliothèques standardes liées automatiquement)
Pour compiler un gros projet, automatisation de la partie ci dessus avec Make
utiliser un fichier MakeFile qui serra appelé par la commande make
gcc -MM *.c
va afficher l'integralité des dépendances de chaque fichier .c dans le repertoire courant
all : executableRealiseParDefaultAlappelDeMake
executableSouhaité : lister tout les .o requis
commande pour realiser la liaison -librarieExternes
## liste de dépendence :
copier le resultat de gcc -MM et intercaller les commandes pour compiler en objet
redaction du make file
Attention : commencer chaque ligne de commande par une tabulation et pas un espace
Doxygen permet de réaliser la documentation technique pour du code C et C++
Documentation officielle
superbe documentation doxygen
doxygen -v
doxygen Doxyfile
generer la documentation du projet
Ecrire la documentation du code en entête du programme, entre /** et */
ou en commentaire de méthodes, énumeration, structure ...
De nombreuses options sont disponibles pour la documentation :
  • \date
  • \author
  • \version
  • \brief (pour une courte description)
  • \def (pour une macro ou constante)
  • \param
  • \return
